# 販賣部結算與盤點系統

一個免安裝、可裝到手機當 App 用的販賣部系統。資料存在你自己的裝置上。

## 功能

- 🛒 **結算收銀** — 點選商品加入結算、自動算總額、完成後自動扣庫存並記錄營業額
- 📦 **商品管理** — 新增 / 編輯 / 刪除商品(名稱、售價、成本、庫存、分類、條碼)
- 📋 **庫存盤點** — 可**用相機掃 QR 碼**自動帶出商品、輸入實際數量;也可手動輸入。自動顯示與帳面庫存的差異(盈虧),確認後一鍵更新庫存
- 📱 **QR 碼標籤** — 每個商品可產生 QR 碼,並能**整批列印**貼到商品 / 貨架上,盤點時掃描超快
- 📊 **報表與紀錄** — 今日營業額 / 筆數 / 售出件數 / 預估毛利、交易明細、盤點歷史,可匯出 CSV
- 📑 **Excel 報表** — 一鍵匯出含「商品清單 / 盤點表 / 營業報表」三分頁的 .xlsx;盤點表自動算差異、營業報表自動加總(內建 Excel 公式)
- 💾 **備份 / 還原** — 一鍵下載備份檔,換手機或重灌可還原
- ☁️ **雲端同步** — 用「店家代碼 + 密碼」把資料存到雲端(Supabase);手機上傳、電腦下載,跨裝置搬資料

> 首次開啟會自動帶入 5 筆示範商品(可樂、礦泉水…),你可以直接編輯或刪除換成自己的商品。

## 怎麼在電腦上打開

直接用瀏覽器打開資料夾裡的 `index.html` 即可使用(Chrome / Edge 建議)。

## 怎麼裝到手機當 App 用(PWA)

手機要安裝成 App,網頁必須透過網址(http/https)開啟,不能用傳檔的方式。下面任選一種:

### 方法 A：用電腦開個小伺服器,手機連同一個 Wi-Fi 連線(最簡單)

1. 電腦和手機連到**同一個 Wi-Fi**。
2. 在這個資料夾開啟終端機(PowerShell),執行下面其中一個:

   - 有 Python：
     ```
     python -m http.server 8000
     ```
   - 有 Node.js：
     ```
     npx serve -l 8000
     ```

3. 查出電腦的區網 IP(PowerShell 執行 `ipconfig`,找 `IPv4 位址`,例如 `192.168.1.20`)。
4. 手機瀏覽器輸入：`http://192.168.1.20:8000`
5. 打開後:
   - **Android(Chrome)**：右上選單 → 「加到主畫面 / 安裝應用程式」
   - **iPhone(Safari)**：下方分享鈕 → 「加入主畫面」
6. 主畫面就會出現「販賣部系統」圖示,點開即全螢幕、可離線使用。

> 注意：用這個方法時,資料存在**手機瀏覽器本機**。電腦只是負責第一次把程式送到手機,之後手機離線也能用。

### 方法 B：放到免費網路空間(隨時隨地可用)

把這個資料夾上傳到 GitHub Pages / Netlify / Cloudflare Pages 之類的免費靜態空間,得到一個網址後,用手機開啟再「加入主畫面」即可。需要的話我可以教你一步步上傳。

### 之後想包成可安裝的 APK?

目前是 PWA。等你用順了,可以再用 Capacitor 之類的工具把同一份程式包成 Android 的 `.apk`,程式幾乎不用重寫。需要時再跟我說。

## QR 碼盤點怎麼用

1. **先印 QR 標籤**:到「商品」頁 → 按「🖨 列印 QR 標籤」一次印出所有商品的 QR,剪下貼到對應商品 / 貨架。
   (也可在單一商品的編輯視窗按「顯示此商品 QR 碼」單獨列印。)
2. **盤點時掃描**:到「盤點」頁 → 按「📷 掃描 QR 碼盤點」→ 對準商品的 QR →
   畫面自動顯示商品名稱 → 輸入實際清點到的數量 → 按「確認並繼續」→ 換下一個商品繼續掃。
3. 全部掃完按「關閉」,回到盤點頁會看到剛剛輸入的數量與差異,最後按「✓ 完成盤點並更新庫存」。

> ⚠️ **相機只能在 https 或 localhost 下使用**(瀏覽器安全限制)。所以掃描功能要用「方法 A」開伺服器或「方法 B」放到網路空間後才能掃;直接雙擊 `index.html`(file://)無法開相機,但其他功能都正常。
>
### 商品代碼(QR 下方的 P001…)

- 每個商品都有一個**固定不變的「商品代碼」**(例如 `P001`),會印在 QR 碼正下方。
- **QR 碼編的是這個代碼,不是商品名稱**。所以以後商品換名稱、改價格、換包裝,**QR 碼都不用重印**——只要代碼不變,掃了一樣對到同一個商品。
- 新增商品時會自動帶入下一個代碼(P001、P002…),你也可以自己改成好記的代碼(代碼不可重複)。
- 編碼優先順序:**商品代碼 > 條碼 > 內部編號**。之前用舊版印的 QR(`PID:xxx`)仍然可以掃。

## 雲端同步怎麼用(手機 ↔ 電腦搬資料)

到「報表」頁最下面的 **☁️ 雲端同步**:

1. 自訂一組**店家代碼**(例:`fanbu2026`)和**密碼**,兩台裝置都用同一組。
2. **手機**:整理好商品/盤點資料後 → 按「☁️⬆ 上傳雲端」。
3. **電腦**(或另一支手機):輸入相同的代碼與密碼 → 按「☁️⬇ 下載雲端」,本機資料會被雲端的覆蓋。
4. 代碼與密碼會記在該裝置,下次自動帶入。

> ⚠️ 這是**手動**同步(不是自動即時):誰先上傳,雲端就是誰的版本;下載會覆蓋本機。建議固定由一台當「主機」上傳,其他裝置只下載,避免互相蓋掉。
> 🔑 **代碼和密碼請記好,忘記無法救回**雲端資料。

## 重要提醒

- **資料只存在這台裝置的瀏覽器**:換裝置、清除瀏覽器資料會不見,請定期用「報表 → 備份資料」下載備份檔。
- 多台裝置目前**不會自動同步**(你選的是本機儲存)。若日後需要多人多裝置共用同一份資料,再跟我說,可改用雲端資料庫。
